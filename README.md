# 📦 Product Data Source

For this project, product data is served from a **local JSON file** instead of using external APIs like **DummyJSON** or **FakeStore API**.

### Reason

The objective of this project was to focus on **frontend architecture, UI implementation, and application logic** rather than backend integration.

Using a local `.json` file allowed:

- Full control over product data structure
- Faster development and testing
- Consistent data during UI development
- Simulating API responses without relying on third-party services

This approach ensures the application remains **self-contained and fully functional without external dependencies**.

If needed, the project can easily be connected to a **real backend API** in future iterations.
